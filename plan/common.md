# phase-ai-anime-skill 通用改造约束

本文件是本仓库全部 phase 的长期稳定约束来源。任何 phase 执行都必须把本文件作为上下文，而不是只看局部任务。

## 结论

本仓库要从长篇小说 phase workflow 改造成 AI 漫剧长线制作 Skill。核心不是承诺直接生成热视频，而是提供稳定、简单、可续跑、可插拔的制作框架，让用户从剧情设计、台词、分镜、prompt、音频时间轴、生成任务到最终合成清单能从 0 到 1 跑通。

## 长期硬约束

- 面向 AI 漫剧小白，文档必须清晰、可执行、少术语堆叠。
- phase 颗粒度要小，剧情、台词、分镜、prompt、音频、生成任务、合成不得混成一阶段。
- 音画同步必须落在 timeline/shot specs 中，不得只在自然语言说明里表达。
- generation job specs 必须 provider-neutral；真实调用模型前必须确认 provider、鉴权、成本和素材边界。
- 工作流节点必须可插入、删除、替换，但要保留输入输出契约。
- `src` 工程体系负责可运行 MVP，不把可执行逻辑散落在文档里。
- 不写入 API key、token、cookie、本地私密素材路径。
- 角色一致性、风格一致性和可合成性优先于单镜头惊艳。
- 发布、打 tag、归档、公开传播、商用素材授权由人类决定。

## 非目标

- 本轮不真实调用任何外部图像、视频、TTS 或音乐模型。
- 本轮不承诺输出最终可公开视频，只输出可运行 MVP 框架与生产蓝图。
- 本轮不把工作流绑定到某一家模型或云服务。
