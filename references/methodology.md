# AI 漫剧长线工作流方法论（Phase-AI-Anime Workflow）

AI 漫剧不是“把小说 prompt 丢给视频模型”。它是一个多模态制作系统：剧作负责让人想看，分镜负责让模型知道该拍什么，音频时间轴负责让画面有节奏，生成任务规格负责让不同模型能协同，最终合成负责把素材变成可传播视频。

## 1. 为什么需要 phase 化

AI 漫剧长线制作最常见的失稳点有六类：

1. **角色漂移**：同一个角色在不同镜头里脸、服装、年龄、体态、声口不一致。
2. **风格漂移**：画面从二次元变成写实，或色彩、线条、光影逐镜头变样。
3. **镜头断裂**：上一镜头的空间关系、动作方向、道具位置无法接到下一镜头。
4. **音画错位**：台词长度、停顿、口型、动作和字幕时间不在同一条节拍上。
5. **模型耦合**：工作流只适配某一家工具，换 provider 后所有 prompt 和产物都失效。
6. **发布前失焦**：创作时觉得每个镜头都不错，合成后却没有钩子、节奏或转发点。

Phase-AI-Anime 的核心主张是：

> 把需要稳定的制作秩序从模型记忆中剥离出来，外部化为仓库里的制作合同；AI 只在当前小窗口完成当前合同。

## 2. 当代 AI 漫剧的前沿共性

根据当前主流视频、图像和语音工具的能力演进，稳定工作流应围绕这些能力设计，而不是绑定具体品牌：

- **参考图与角色一致性**：用角色板、服装板、表情板、关键姿态和风格参考图作为下游视频镜头的输入锚。
- **首尾帧与镜头延展**：把镜头拆成可控短片段，用 first frame / last frame / extend shot 维持连续性。
- **镜头与运动控制**：prompt 中显式写 camera move、framing、subject action、motion path，避免“好看但不可剪”。
- **原生音频与后期音频并存**：有的模型能生成音频，但可控对白仍应保留 TTS/SFX/music 的独立时间轴，方便替换和精修。
- **多模型协作**：专业流程会在一个项目里混用图像、视频、音频、字幕、剪辑和人工审查，不应把全部风险压到单个模型。
- **短剧传播语法**：前三秒必须清楚“谁遇到什么反差/危险”，中段每 6-10 秒要有信息变化，结尾留一个具体未解问题或情绪牵引。

## 3. 好看的 AI 漫剧靠什么活

### 3.1 钩子不是噱头，是观看契约

一个可传播的开场通常同时满足：

- 画面上立刻可识别：角色、处境、反差、危险至少出现两项。
- 信息上立刻提问：观众知道“我接下来要等哪个答案”。
- 情绪上立刻站队：观众愿意支持、担心、吐槽或好奇某个角色。

### 3.2 角色先于设定

AI 漫剧尤其依赖“可重复角色资产”。每个核心角色至少要落盘：

- 外观锚点：发型、脸型、服装、配饰、体态、代表色。
- 行为锚点：常用动作、表情习惯、进入镜头的方式。
- 声口锚点：句长、口癖、情绪温度、停顿习惯。
- 冲突锚点：想要什么、害怕什么、被谁或什么阻挡。

### 3.3 分镜是模型友好语言

文字剧情不能直接驱动视频模型。镜头卡必须把故事翻译成：

- `shot_id`、时长、画幅、镜头类型。
- 主体、动作、情绪、道具和空间关系。
- camera move、转场、首尾帧意图。
- 台词、SFX、音乐、字幕与动作的同步点。
- image prompt、video prompt、negative prompt 和参考资产入口。

### 3.4 音画同步要先设计

不要等视频生成后再“贴声音”。每个镜头先规划：

- 台词字符数与预计语速。
- 情绪标签和 TTS 方向。
- 动作命中点：门响、回头、眨眼、掏出物件、转场。
- 字幕出现、消失和换行位置。
- 音乐进入、压低、停顿、爆点和余韵。

## 5. 最小可行 phase 图

1. `phase-0-concept-promise`
2. `phase-1-cast-style-bible`
3. `phase-2-episode-beat-sheet`
4. `phase-3-dialogue-voice-script`
5. `phase-4-shot-storyboard`
6. `phase-5-visual-prompt-pack`
7. `phase-6-audio-timeline`
8. `phase-7-generation-job-specs`
9. `phase-8-assembly-qc`

这条链故意小颗粒：每个 phase 的产物都能被人工检查，也能被下游模型或工具消费。

## 6. 节点可插拔模型

每个工作流节点必须声明：

- `inputs`：需要哪些上游制品。
- `outputs`：写出哪些文件或任务规格。
- `requiredArtifacts`：完成检查至少看什么。
- `replaceableBy`：可替换 provider 或人工步骤。
- `failureMode`：失败时会污染哪个下游环节。

新增节点时，只能插在输入已满足、输出被下游消费的位置。删除节点时，必须确认没有下游 phase 依赖它的输出。

## 7. 质量检查口径

- 剧情：前三秒是否给出明确问题，结尾是否留下具体牵引。
- 台词：每句是否可被角色身份解释，是否能被 TTS 自然读出。
- 分镜：每个镜头是否能独立生成，镜头之间是否有空间/动作连续性。
- Prompt：是否包含主体、动作、情绪、场景、光线、镜头、风格和禁区。
- 音频：台词、SFX、音乐、停顿是否在时间轴上覆盖全部镜头。
- 合成：assembly manifest 是否能列出所有素材路径、时长、字幕和缺口。

## 8. MVP 边界

MVP 只负责把工作流跑通到“可生成任务清单”：不代用户选择商用模型，不存储密钥，不承诺直接产出公开视频。真实调用 provider 是后续 adapter phase，可以独立插入。
